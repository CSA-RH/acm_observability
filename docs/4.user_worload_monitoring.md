# User Workload Monitoring

OpenShift User Workload Monitoring (UWM) allows applications to expose custom Prometheus metrics via `ServiceMonitor` and `PodMonitor` CRs. These metrics can be pushed from managed clusters to the hub's Thanos for centralized dashboarding and alerting.

The method to forward UWM metrics depends on which metrics collection stack is active (standard or MCOA). Both are mutually exclusive.

- **[4.1 UWM with Standard Metrics-Collector](4.1.uwm-standartmc-configuration_and_demo.md)** : Uses the standart metrics collector to scrap metrics from the managed cluster and push them to the Hub.
- **[4.2 UWM with MCOA](4.2.uwm-mcoamc-configuration_and_demo.md)** : Uses the MCOA metrics collector to scrap metrics from the managed cluster and push them to the Hub.
