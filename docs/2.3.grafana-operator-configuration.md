
# Grafana Operator : Integration with ACM Observability Stack

This document covers the installation and configuration of the community Grafana Operator to connect an external, Grafana instance to the ACM multicluster observability stack. It includes creating a datasource pointing to the ACM Thanos Query Frontend, deploying declarative dashboards via CRDs, and configuring authentication with the `observability-grafana-certs` secret.

> **Note:** This is not an officially supported Red Hat integration because Grafana Operator is a **community operator**, so falls outside the ACM support scope.

## Understanding the Support Scope

Before diving into the technical steps, it is vital to understand what is (and isn't) supported by Red Hat when using an external Grafana instance:

- **Supported**: The connectivity to the Observatorium API and the health of the underlying metrics pipeline.
- **Not Supported**: The configuration, maintenance, or troubleshooting of external Grafana dashboards. If you choose to use your own Grafana instance, the responsibility for dashboard persistence and visualization logic remains with the user.
- **Security Note**: The certificates extracted for this connection grant read-only access to metrics.

Reference: [ch-stark/externalgrafanablog : Connecting External Grafana to RHACM](https://github.com/ch-stark/externalgrafanablog/blob/main/fetchfromgrafana.md)

## Prerequisites

- MCO installed and running on the hub cluster
- `oc` and `jq` installed
- Cluster-admin access

---

## 1. Install the Grafana Operator

Install from OperatorHub (community operators):

```bash
oc apply -f - <<'EOF'
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: grafana-operator
  namespace: openshift-operators
spec:
  channel: v5
  installPlanApproval: Automatic
  name: grafana-operator
  source: community-operators
  sourceNamespace: openshift-marketplace
EOF
```

Wait for the operator to be ready:

```bash
oc -n openshift-operators get deployment | grep grafana-operator
# The deployment name may vary (e.g. grafana-operator-controller-manager-v5)
oc -n openshift-operators wait --for=condition=Available deployment/grafana-operator-controller-manager-v5 --timeout=120s
```

## 2. Create a namespace for the Grafana instance

```bash
oc create namespace grafana-operator
```

## 3. Extract ACM Observability connection credentials

The external Grafana connects to the ACM Thanos via the Observatorium API route using mTLS. You need four pieces of information from the ACM observability stack: the API URL, the CA certificate, the client certificate, and the client key.

Script from [ch-stark/externalgrafanablog](https://github.com/ch-stark/externalgrafanablog/blob/main/fetchfromgrafana.md):

```bash
# Create a temporary file to store the details
touch /tmp/external-grafana.txt

echo "# --- RHACM Observability Connection Details ---" > /tmp/external-grafana.txt

# 1. Fetch the Observatorium API URL
host=$(oc -n open-cluster-management-observability get route observatorium-api -o json | jq -r '.spec.host')
echo "# URL" >> /tmp/external-grafana.txt
echo "https://${host}/api/metrics/v1/default" >> /tmp/external-grafana.txt
echo >> /tmp/external-grafana.txt

# 2. Fetch the CA Certificate
crt=$(oc -n open-cluster-management-observability get secret observability-server-ca-certs -o json | jq -r '.data."ca.crt"' | base64 -d )
echo "# ca.crt" >> /tmp/external-grafana.txt
echo "${crt}" >> /tmp/external-grafana.txt
echo >> /tmp/external-grafana.txt

# 3. Fetch the Client Certificate (tls.crt)
client_crt=$(oc -n open-cluster-management-observability get secret observability-grafana-certs -o json | jq -r '.data."tls.crt"'| base64 -d)
echo "# client.crt" >> /tmp/external-grafana.txt
echo "${client_crt}" >> /tmp/external-grafana.txt
echo >> /tmp/external-grafana.txt

# 4. Fetch the Client Key (tls.key)
client_key=$(oc -n open-cluster-management-observability get secret observability-grafana-certs -o json | jq -r '.data."tls.key"' | base64 -d)
echo "# client.key" >> /tmp/external-grafana.txt
echo "${client_key}" >> /tmp/external-grafana.txt

echo "Extraction complete. View details at /tmp/external-grafana.txt"
```

Verify the extracted details:

```bash
cat /tmp/external-grafana.txt
```

Create a secret in the Grafana namespace with these credentials:

```bash
oc -n grafana-operator create secret generic acm-observability-certs \
  --from-file=ca.crt=<(echo "$crt") \
  --from-file=tls.crt=<(echo "$client_crt") \
  --from-file=tls.key=<(echo "$client_key")
```

Reference: [ch-stark/externalgrafanablog](https://github.com/ch-stark/externalgrafanablog/blob/main/fetchfromgrafana.md)

## 4. Create the Grafana instance

> **Note:** The admin credentials must be set directly in the `config.security` section. The Grafana Operator reads these values to authenticate against the Grafana API. Using environment variable references (e.g. `${GF_SECURITY_ADMIN_USER}`) in the config will fail because the operator interprets them as literal strings.

```bash
oc apply -f - <<'EOF'
apiVersion: grafana.integreatly.org/v1beta1
kind: Grafana
metadata:
  name: grafana-operator
  namespace: grafana-operator
  labels:
    dashboards: "grafana-operator"
spec:
  config:
    log:
      mode: "console"
    auth:
      disable_login_form: "false"
    security:
      admin_user: admin
      admin_password: grafana-operator-admin
  deployment:
    spec:
      replicas: 1
      template:
        spec:
          containers:
            - name: grafana
              resources:
                requests:
                  cpu: 100m
                  memory: 256Mi
                limits:
                  cpu: 500m
                  memory: 512Mi
  route:
    spec:
      tls:
        termination: edge
EOF
```

Wait for the Grafana instance to be ready:

```bash
# Check that the Grafana pod is Running:
oc -n grafana-operator get pods

# Verify the Grafana CR status (look for message: "success"):
oc -n grafana-operator get grafana grafana-operator
```

Once ready, get the URL and credentials:

```bash
GRAFANA_URL=$(oc -n grafana-operator get route grafana-operator-route -o jsonpath='{.spec.host}')
echo "Grafana Operator URL: https://$GRAFANA_URL"
echo "Login: admin / grafana-operator-admin"
```

## 5. Create the GrafanaDatasource pointing to ACM Thanos

This datasource uses mTLS to authenticate against the Observatorium API : the same credentials that ACM's internal Grafana uses (from `observability-grafana-certs` and `observability-server-ca-certs` secrets).

Declarative approach using Grafana Operator CRs:

```bash
OBSERVATORIUM_URL="https://${host}/api/metrics/v1/default"

cat <<EOF | oc apply -f -
apiVersion: grafana.integreatly.org/v1beta1
kind: GrafanaDatasource
metadata:
  name: acm-observatorium
  namespace: grafana-operator
spec:
  datasource:
    name: ACM Observatorium
    type: prometheus
    access: proxy
    url: "${OBSERVATORIUM_URL}"
    isDefault: true
    editable: true
    jsonData:
      tlsAuth: true
      tlsAuthWithCACert: true
      timeInterval: 5m
    secureJsonData:
      tlsCACert: '\${ca.crt}'
      tlsClientCert: '\${tls.crt}'
      tlsClientKey: '\${tls.key}'
  instanceSelector:
    matchLabels:
      dashboards: "grafana-operator"
  valuesFrom:
    - targetPath: secureJsonData.tlsCACert
      valueFrom:
        secretKeyRef:
          name: acm-observability-certs
          key: ca.crt
    - targetPath: secureJsonData.tlsClientCert
      valueFrom:
        secretKeyRef:
          name: acm-observability-certs
          key: tls.crt
    - targetPath: secureJsonData.tlsClientKey
      valueFrom:
        secretKeyRef:
          name: acm-observability-certs
          key: tls.key
EOF
```

> **Note : `timeInterval: 5m`:** ACM collects metrics every ~5 minutes. The `timeInterval` (Grafana's "Scrape interval") must match this collection interval. If set too low (e.g. `5s`), Grafana variables like `$__rate_interval` resolve to values too short for the available data resolution, causing `rate()` queries to return no data. Setting `timeInterval: 5m` ensures `$__rate_interval` produces a window large enough for at least 2 data points.

Verify the datasource is synced:

```bash
oc -n grafana-operator get grafanadatasource acm-observatorium -o jsonpath='{.status}'
```

## Test queries

After logging into the Grafana Operator UI, go to **Explore** (compass icon), select the **ACM Observatorium** data source, and run these queries to verify the integration:

```promql
# Memory available per cluster:
:node_memory_MemAvailable_bytes:sum

# Pod count per cluster and namespace:
sum by (cluster, namespace) (kube_pod_info)
```
