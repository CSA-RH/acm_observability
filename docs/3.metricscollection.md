# Custom Metrics, Alerts, and Recording Rules

ACM provides two methods to collect and forward metrics from the managed/spoke clusters, to the hub. Both approaches are **mutually exclusive** and the choice is **fleet-wide**: enabling MCOA on the hub switches all managed clusters from the standard `metrics-collector` to the `PrometheusAgent`.

| Method | Description |
|---|---|
| **Standard metrics-collector** | Uses the `metrics-collector` and the `observability-metrics-custom-allowlist` ConfigMap to select which metrics to forward. |
| **MCOA (Multicluster Observability Add-on)** | Uses `PrometheusAgent` to push metrics to the Hub Observability API, `ScrapeConfig` and `PrometheusRule` CRs for metrics selection and local recording rules. |

Hub-side components (Thanos Receive, Thanos Ruler, Alertmanager, Grafana) and the `thanos-ruler-custom-rules` ConfigMap for hub-side alerts and recording rules are the same regardless of which collection method is used.
