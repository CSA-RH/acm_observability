
# Install and Configure Perses Dashboards

## Disclaimer
Perses dashboards are currently a **Technology Preview** feature and is not supported for production use.

## Introduction

This document covers the installation of the Cluster Observability Operator (COO), enabling Perses via the UIPlugin CR, creating a global datasource pointing to ACM's Thanos Query Frontend, and deploying custom Perses dashboards with multicluster metrics.

Perses is delivered via the COO: a separate operator from MCO. Both coexist on the hub cluster.

## 1. Install the Cluster Observability Operator

Install COO from OperatorHub:

```bash
oc apply -f - <<'EOF'
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: cluster-observability-operator
  namespace: openshift-operators
spec:
  channel: stable
  installPlanApproval: Automatic
  name: cluster-observability-operator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
EOF

# Wait for the operator to be ready
oc -n openshift-operators wait --for=condition=Available deployment -l app.kubernetes.io/name=observability-operator --timeout=120s
```

## 2. Enable Perses via the UIPlugin

```bash
oc apply -f - <<'EOF'
apiVersion: observability.openshift.io/v1alpha1
kind: UIPlugin
metadata:
  name: monitoring
spec:
  type: Monitoring
  monitoring:
    perses:
      enabled: true
EOF
```

Wait for it to reconcile:

```bash
# Wait until STATUS shows Available
oc get uiplugin monitoring -o jsonpath='{range .status.conditions[*]}{.type}: {.status} ({.reason}){"\n"}{end}'
```

After reconciliation, COO deploys:
- The Perses Operator
- `PersesDashboard`, `PersesDatasource`, `PersesGlobalDatasource` CRDs
- RBAC ClusterRoles (`persesdashboard-editor-role`, `persesdashboard-viewer-role`, etc.)

In the OpenShift console: **Observe > Dashboards (Perses)** should now be visible.

## 3. Create a PersesGlobalDatasource pointing to MCO's Thanos

To make Perses dashboards query the multicluster metrics collected by MCO, create a datasource pointing to the Thanos Query Frontend:

```bash
oc apply -f - <<'EOF'
apiVersion: perses.dev/v1alpha2
kind: PersesGlobalDatasource
metadata:
  name: acm-thanos-querier
spec:
  config:
    display:
      name: "ACM Thanos Querier (Multicluster Metrics)"
    default: true
    plugin:
      kind: "PrometheusDatasource"
      spec:
        proxy:
          kind: HTTPProxy
          spec:
            url: http://observability-thanos-query-frontend.open-cluster-management-observability.svc:9090
EOF
```

## 4. Perses Dashboard Tests

### Test 4.1 : Create a custom dashboard using the CLI

Create a `PersesDashboard` CR that shows multicluster node memory usage:

```bash
oc apply -f - <<'EOF'
apiVersion: perses.dev/v1alpha2
kind: PersesDashboard
metadata:
  name: acm-demo-cluster-health
  namespace: open-cluster-management-observability
spec:
  config:
    display:
      name: "ACM Demo : Cluster Health Overview"
    variables:
      - kind: ListVariable
        spec:
          name: cluster
          display:
            name: Cluster
          allowMultiple: true
          allowAllValue: true
          plugin:
            kind: PrometheusLabelValuesVariable
            spec:
              labelName: cluster
              matchers:
                - up
    panels:
      nodeMemory:
        kind: Panel
        spec:
          display:
            name: "Node Memory Available (by cluster)"
          plugin:
            kind: TimeSeriesChart
            spec: {}
          queries:
            - kind: TimeSeriesQuery
              spec:
                plugin:
                  kind: PrometheusTimeSeriesQuery
                  spec:
                    query: ':node_memory_MemAvailable_bytes:sum{cluster=~"$cluster"}'
      nodeCPU:
        kind: Panel
        spec:
          display:
            name: "Node CPU Utilization (by cluster)"
          plugin:
            kind: TimeSeriesChart
            spec: {}
          queries:
            - kind: TimeSeriesQuery
              spec:
                plugin:
                  kind: PrometheusTimeSeriesQuery
                  spec:
                    query: '1 - avg by (cluster) (rate(node_cpu_seconds_total{mode="idle", cluster=~"$cluster"}[10m]))'
      podCount:
        kind: Panel
        spec:
          display:
            name: "Running Pods (by cluster)"
          plugin:
            kind: TimeSeriesChart
            spec: {}
          queries:
            - kind: TimeSeriesQuery
              spec:
                plugin:
                  kind: PrometheusTimeSeriesQuery
                  spec:
                    query: 'count by (cluster) (kube_pod_info{cluster=~"$cluster"})'
    layouts:
      - kind: Grid
        spec:
          display:
            title: "Cluster Health"
            collapse:
              open: true
          items:
            - x: 0
              y: 0
              width: 24
              height: 6
              content:
                "$ref": "#/spec/panels/nodeMemory"
            - x: 0
              y: 6
              width: 24
              height: 6
              content:
                "$ref": "#/spec/panels/nodeCPU"
            - x: 0
              y: 12
              width: 24
              height: 4
              content:
                "$ref": "#/spec/panels/podCount"
    duration: 1h
EOF
```

Verify:

```bash
oc get persesdashboard -n open-cluster-management-observability
# Expected: acm-demo-cluster-health
```

Open the console: **Observe > Dashboards (Perses)** : select the `open-cluster-management-observability` project and the dashboard should appear.

### Test 4.2 : Dashboard RBAC

Grant a non-admin user read-only access to Perses dashboards in a specific namespace:

```bash
oc apply -f - <<'EOF'
kind: RoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: demo-user-dashboard-viewer
  namespace: open-cluster-management-observability
subjects:
  - kind: User
    apiGroup: rbac.authorization.k8s.io
    name: demo-user
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: persesdashboard-viewer-role
---
kind: RoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: demo-user-datasource-viewer
  namespace: open-cluster-management-observability
subjects:
  - kind: User
    apiGroup: rbac.authorization.k8s.io
    name: demo-user
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: persesdatasource-viewer-role
EOF
```

The user `demo-user` can now see dashboards in `open-cluster-management-observability` but cannot edit or delete them.
