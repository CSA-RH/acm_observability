# Custom Metrics, Alerts, and Recording Rules

ACM provides two methods to collect and forward metrics from managed (spoke) clusters to the hub. Both approaches are **mutually exclusive**: you must choose one per managed cluster.

| Method | Description |
|---|---|
| **Standard (legacy) metrics-collector** | Uses the `metrics-collector` binary and the `observability-metrics-custom-allowlist` ConfigMap to select which metrics to forward. |
| **MCOA (Multicluster Observability Add-on)** | Uses `PrometheusAgent` with `ScrapeConfig` and `PrometheusRule` CRs for metrics selection and local recording rules. |

Hub-side components (Thanos Receive, Thanos Ruler, Alertmanager, Grafana) and the `thanos-ruler-custom-rules` ConfigMap for hub-side alerts and recording rules are the same regardless of which collection method is used.
