
# Grafana Developer Instance : Installation and Demos

This document covers the deployment of the `grafana-dev` instance, which provides a full Grafana UI for designing custom dashboards connected to the ACM observability data source. It walks through installing the dev instance, creating and editing dashboards using the visual editor, exporting them as ConfigMaps, and applying those ConfigMaps to the hub cluster so that the default Grafana (managed by MCO) picks up the custom dashboards.

> **Note:** The `grafana-dev` instance is an auxiliary tool to help design dashboards visually using the Grafana UI. It is not strictly required, dashboards can be created from scratch as ConfigMaps and applied directly to the cluster without ever deploying `grafana-dev`. However, writing Grafana dashboard JSON by hand is significantly harder and error-prone compared to using the visual editor.

## Prerequisites

- MCO must be installed and running (the `MultiClusterObservability` CR must be applied.

---

## 1. Install grafana-dev

Clone the repository and run the setup script:

```bash
git clone https://github.com/stolostron/multicluster-observability-operator.git
cd multicluster-observability-operator/tools
./setup-grafana-dev.sh --deploy
```

Wait for the pod:

```bash
oc -n open-cluster-management-observability wait --for=condition=Available deployment/grafana-dev --timeout=120s
```

Get the URL:

```bash
GRAFANA_DEV_URL=$(oc -n open-cluster-management-observability get route grafana-dev -o jsonpath='{.spec.host}')
echo "Grafana-dev URL: https://$GRAFANA_DEV_URL"
```

**1.1. Log in to the Grafana UI using OpenShift credentials**

**1.2. Switch your user to a Grafana admin**

By default, OpenShift OAuth users get the **Viewer** role in Grafana : they can see dashboards and explore data but cannot create, edit, or save anything. This script promotes the user to **Grafana Admin**, enabling full dashboard creation and editing:

> **Important:** You must first **log in** to grafana-dev via the browser (so the user account is created in Grafana's internal database), and **then** run the script. If you run the script before logging in, there is no user to promote and it will fail.

```bash
./switch-to-grafana-admin.sh <user_name>
```

Expected output:

```
User <user_name> switched to be grafana admin
```

After refreshing the browser (or restarting it), the **+** (Create) icon should appear. In Grafana 12.x, this button is in the **top-right** of the UI (not in the left sidebar as in older versions). Without this step, you will not see the Create option and cannot build dashboards.

## 2. Capabilities

**What you can do:**
- Create custom dashboards: full Grafana editor (time series, stat, gauge, table, heatmap, etc.)
- Write PromQL queries against the multicluster metrics via the `Observatorium` datasource
- Export dashboards as ConfigMaps
- Organize dashboards into custom folders via ConfigMap annotations
- Create dashboards directly as ConfigMaps (GitOps/ArgoCD easiness)

Alerting and recording rules are configured separately via the `thanos-ruler-custom-rules` ConfigMap, not through Grafana.

---

## 3. Demo: Create a custom multicluster dashboard

1. Click the **+** icon > **New Dashboard** > **Add new panel**
2. Click the **+ Add visualization** button
3. Select the data source `Observatarium default`
4. In the **Query** tab, select `Observatorium` from the data source selector
5. Enter this PromQL:

```
:node_memory_MemAvailable_bytes:sum
```

6. Set **Legend** to `{{cluster}}`
7. Unit: **Data > bytes(IEC)** : displays values as GiB
8. Panel title: `Memory Available by Cluster`
9. Click **Save** icon in the dashboard header, add a descriptive name, click **Save**

> **Note:** Use **Code** mode (not Builder) in the query editor : the Builder mode can break complex PromQL expressions. ACM only collects recording rules (pre-aggregated metrics), not raw metrics like `node_memory_MemTotal_bytes`. Use the recording rules prefixed with `:` or `cluster:`.

Add a second panel (CPU):

1. Click **Add panel** > **Add new panel**
2. Select `Observatorium` datasource, switch to **Code** mode, PromQL:

```
cluster:cpu_usage_cores:sum
```

3. Legend: `{{cluster}}`
4. Unit: **Misc > short** : displays values as plain numbers (e.g., `2.3` means 2.3 cores in use)
5. Panel title: `CPU Usage Cores by Cluster`
6. Apply and save

## 4. Demo: Export the dashboard from Demo 4 to a ConfigMap

Export the dashboard created in Demo 3 (with the Memory and CPU panels) to a ConfigMap so it appears in the production (read-only) Grafana.

**4.1. Get the dashboard name:**

In grafana-dev, confirm the dashboard name. If you followed Demo 4, it should be something like `my_metrics` or whatever name you chose when saving.

**4.2. Export using the script:**

```bash
./generate-dashboard-configmap-yaml.sh "my_metrics"
```

Replace `my_metrics` with the exact dashboard name you used. Expected output:

```
Save dashboard <my-metrics> to ./my-metrics.yaml
```
